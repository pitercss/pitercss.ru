Декларативное тестирование верстки

		{
			name: 'Viacheslav',

Какие проблемы сопутствуют тестированию верстки?

  • Знание минимум одного языка программирования
  • Нехватка времени и / или желания
  • Тестирование вручную
  • Настройка инфраструктуры
  • Заказчик не понимает ценности тестирования как такового

А если есть желание, время, да и заказчик молодец?

Наиболее распространенное решение - Selenium

  • Взаимодействие с DOM деревом
  • Клики по элементам
  • Ожидание асинхронного контента
  • Ввод текстовых данных
  • Отправка форм
  • Запуск JavaScript в браузере кода при необходимости

А можно ли проще?

Да, с помощью Galen.

Работает на основе Selenium и предоставляет базовую функциональность по тестированию верстки в декларативном стиле, позволяя:

  • Проверять размеры элементов
  • Проверять расположение элементов относительно друг друга
  • Проверять цветовые гаммы
  • Создавать собственные правила
  • Генерировать детальные отчеты в html формате

Подводя итоги

  • Тестировать верстку можно и нужно
  • Это не так сложно как кажется
  • Тесты поддерживаемы (импорты) и читаемы (благодаря собственным правилам)
  • Выгода очевидна при необходимости подержки n браузеров с k вариантами разрешений
